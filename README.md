## Project Overview

This Streamlit application is the **final delivery (Milestone 4)** of a four-milestone project to build a
**Cyber Threat Intelligence (CTI) platform for Global Financial Institutions (GFI)**.
The platform targets major financial institutions operating investment banking,
capital markets, and retail banking — facing threat actors from ransomware groups
to nation-state APTs.

**Milestone 4** builds on all prior milestones and adds:

1. **Intelligence Summary** — 5 severity-rated findings with TTPs, implications, and source attribution
2. **Operational Triage Dashboard** — Severity-ranked alert queue with filters, recommended actions, and CSV/JSON/STIX 2.1 export
3. **Role-Based Views with LLM Integration** — Executive and Analyst briefs generated by Google Gemini Flash from live platform data
4. **Dissemination Strategy** — Stakeholder communication matrix, IOC-to-control mapping, courses of action
5. **Future CTI Platform Directions** — 3 justified roadmap items with phased timelines

## Quick Start (Local)

### 1. Install dependencies

```bash
pip install -r requirements.txt
```

### 2. Run the app

```bash
streamlit run milestone4_app.py
```

The app will open in your browser at `http://localhost:8501`.

All data is fetched live — internet access required. API calls are cached 30-120 minutes per source. Hardcoded fallback data loads automatically if any API is unavailable.

### 3. Optional: LLM Integration

Place a Gemini API key in `.streamlit/secrets.toml`:
```
GEMINI_API_KEY = "your_key_here"
```
Without a key, the role-based views use pre-built template briefs.

## Data Sources

All data sources are **free, publicly accessible OSINT** (except VirusTotal and Gemini which require free API keys):

| Source | URL | Type | Auth | Used In |
|---|---|---|---|---|
| CISA KEV | cisa.gov | JSON (GET) | None | Classifier, Dashboard, Triage |
| EPSS | api.first.org | JSON (GET) | None | Classifier (batch query), Triage |
| URLhaus | urlhaus.abuse.ch | CSV (GET) | None | Cross-Source Correlation, Triage |
| MalwareBazaar | bazaar.abuse.ch | JSON (POST) | None | Cross-Source Correlation, Triage |
| ThreatFox | threatfox.abuse.ch | CSV (GET) | None | Cross-Source Correlation, Triage |
| Ransomware.live | api.ransomware.live | JSON (GET) | None | Anomaly Detection, Triage |
| SEC EDGAR 8-K | efts.sec.gov | JSON (GET) | None (User-Agent required) | Data Sources |
| VirusTotal | virustotal.com/api/v3 | JSON (GET) | Free API key | Data Sources |
| Google Gemini | generativelanguage.googleapis.com | JSON (POST) | Free API key | Role-Based Views (M4) |

No external data files are required. All data is fetched live from APIs at runtime and cached via `@st.cache_data(ttl=N)`. Fallback data is hardcoded for all sources to ensure live demo reliability.

## Analytics Methodology (Milestone 3)

### Approach 1: CVE Ransomware-Risk Classification

- **Algorithm:** Random Forest (scikit-learn) with `class_weight="balanced"`
- **Features (13):** EPSS score, EPSS percentile, days in KEV, remediation window, vendor criticality, product frequency, and 7 vulnerability-type keyword flags (RCE, privilege escalation, buffer overflow, auth bypass, deserialization, path traversal, SQL injection)
- **Target:** `knownRansomwareCampaignUse` from CISA KEV (binary: Known vs Unknown)
- **EPSS coverage:** Batch-queries EPSS API for all ~1,579 KEV CVE IDs (100% coverage)
- **Evaluation:** Confusion matrix, precision, recall, F1, ROC-AUC (~0.76), feature importance
- **User controls:** Test size, tree count, max depth, classification threshold

### Approach 2: Temporal Anomaly Detection

- **Method:** Rolling z-score (semi-supervised point anomaly detection)
- **Data:** KEV `dateAdded` aggregated monthly; Ransomware.live daily victim counts
- **User controls:** Z-score threshold, rolling window size

### Approach 3: Cross-Source IOC Correlation

- **Method:** Set intersection of malware family names across 3 abuse.ch feeds
- **Scoring:** Compound Confidence Score (CCS) — 1 (single-source) to 3 (tri-source)
- **Data:** URLhaus tags, ThreatFox families, MalwareBazaar signatures

## Operational Intelligence (Milestone 4)

### Intelligence Summary

5 severity-rated intelligence findings (Critical/High/Medium) with:
- Operational implications for GFI
- MITRE ATT&CK TTP references
- Multi-source attribution

### Triage Dashboard

- Unified alert queue from CISA KEV, EPSS, URLhaus, ThreatFox, MalwareBazaar, and Ransomware.live
- Severity ranking (Critical/High/Medium/Low) based on EPSS scores and source corroboration
- Recommended courses of action per alert
- Export: CSV, JSON, STIX 2.1 bundle

### Role-Based Views

- **Executive Summary:** Non-technical brief for CISO/Board with risk posture, top 3 business risks, budget recommendations
- **Analyst Drill-Down:** Technical brief for SOC team with priority actions, specific CVE IDs, detection methods
- Both views powered by Google Gemini Flash LLM with template fallback

### Dissemination Strategy

- Stakeholder communication matrix (who/when/what/how/TLP)
- IOC-to-control course-of-action mapping with owners and priority levels
- Diamond Model update recommendations
- Next CTI iteration planning (feedback loop)

### Future Directions

Three justified development roadmap items:
1. **Real-Time Streaming & SIEM Integration** — Kafka pipelines, TAXII 2.1 server, Splunk/QRadar push
2. **LLM-Powered Automated Reports & RAG Q&A** — Vector-store retrieval, scheduled stakeholder reports
3. **Automated STIX/TAXII Sharing & FS-ISAC Integration** — Sector-level threat intelligence contribution
